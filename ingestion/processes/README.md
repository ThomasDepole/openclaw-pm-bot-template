# processes/

Drop process documentation, policies, and standards here.

**What belongs here:**
- How your team runs sprints or projects (methodology docs)
- Escalation processes and approval chains
- Definition of Done / acceptance criteria standards
- HR policies and employee handbook sections
- Onboarding documentation
- Quality assurance and testing standards
- Communication norms and meeting cadences
- Any "this is how we do things here" documentation

**Tip:** These docs don't change often, but they're high-value context. They help the agent understand *why* things are done a certain way, not just what's happening.
