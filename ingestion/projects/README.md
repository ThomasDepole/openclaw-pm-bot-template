# projects/

Drop project documentation here.

**What belongs here:**
- Statements of Work (SOWs)
- Project charters and briefs
- Project plans and timelines
- Scope documents
- Change orders and amendments
- Technical specifications and discovery docs
- RAID logs (risks, actions, issues, decisions)

**Tip:** Start with the SOW or project brief — it gives the agent the clearest picture of what the project is, what's in scope, and what success looks like.
